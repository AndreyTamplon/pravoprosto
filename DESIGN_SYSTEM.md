# Право Просто — Дизайн-система

## Концепция

**Тема:** Комикс / Цифровой герой.
Ребёнок — не пассивный ученик, а герой, проходящий миссии по защите себя и других в цифровом мире. Визуальный язык заимствован из комиксов: жирные контуры, панели-фреймы, speech-бабблы, POW/ZAP-стиль обратной связи, полутоновые паттерны.

**Тон:** смелый, энергичный, вдохновляющий. Ребёнок чувствует себя крутым и способным, а не поучаемым.

**Референс-мок:** `mock_ui/variant_c/index.html`

- Тёмные секции (`--dark`) используются для контраста: HUD-бар, hero-секция лендинга, экран завершения миссии
- Каждый курс получает свой акцентный цвет из палитры

## Типографика

### Шрифт
**Baloo 2** (Google Fonts) — округлый, жирный, игривый. Подходит для детской аудитории и комикс-стиля.

```
font-family: 'Baloo 2', cursive, sans-serif;
```

### Шкала

| Уровень | Размер | Вес | Использование |
|---|---|---|---|
| Display | 2.5rem+ | 800 | Заголовки лендинга, "МИССИЯ ВЫПОЛНЕНА" |
| H1 | 1.8–2rem | 800 | Заголовки экранов |
| H2 | 1.4–1.6rem | 700 | Заголовки секций, названия курсов |
| H3 | 1.1–1.2rem | 700 | Подзаголовки, названия уроков |
| Body | 1rem | 500 | Основной текст, описания |
| Body Large | 1.1rem | 500-600 | Текст вопросов в уроке |
| Caption | 0.85rem | 600 | Бейджи, метки, мелкий текст |
| Micro | 0.75rem | 600 | Числа в HUD, таймеры |

### Правила
- Текст кнопок — UPPERCASE, letter-spacing: 0.5px
- Заголовки могут иметь лёгкий наклон (transform: rotate(-1..2deg)) для динамичности
- Текст в speech-бабблах — обычный регистр, line-height: 1.5
- Числовые показатели (XP, streak) — жирные, с цветовым акцентом

## Обводки и тени

Это ключевой элемент комикс-стиля — жирные чёрные обводки и плоские offset-тени.

| Токен | Значение | Использование |
|---|---|---|
| `--border` | `3px solid #1E293B` | Обводка карточек, бабблов, бейджей |
| `--border-thick` | `4px solid #1E293B` | Обводка крупных панелей, кнопок |
| `--shadow-comic` | `5px 5px 0 #1E293B` | Тень карточек, кнопок |
| `--shadow-comic-sm` | `3px 3px 0 #1E293B` | Тень мелких элементов |

### Правила
- Тени всегда плоские (offset без blur) — это и есть комикс-стиль
- При hover тень увеличивается (7px 7px), элемент приподнимается
- При active тень уменьшается (2px 2px), элемент "вдавливается"
- Скругление: `border-radius: 12px` как base

## Полутоновый паттерн (Halftone)

Тонкий паттерн из точек, создающий текстуру печатного комикса.

```css
.halftone {
  background-image: radial-gradient(circle, #1E293B 1px, transparent 1px);
  background-size: 12px 12px;
  opacity: 0.06;
  position: absolute;
  inset: 0;
  pointer-events: none;
}
```

Используется как overlay поверх иллюстраций и Hero-секций. Не мешает чтению, но добавляет фактуру.

## Компоненты

### Кнопки

4 варианта, все с жирной обводкой и comic-shadow:

| Вариант | Фон | Цвет | Когда |
|---|---|---|---|
| `btn-primary` | `--orange` | white | Главный CTA: "НАЧАТЬ МИССИЮ", "ПРОВЕРИТЬ" |
| `btn-teal` | `--teal` | white | Вторичный CTA: "ДАЛЕЕ", "ПРОДОЛЖИТЬ" |
| `btn-dark` | `--dark` | white | Специальные действия |
| `btn-outline` | white | `--dark` | Третичные действия: "К КУРСУ", "НАЗАД" |

**Состояния:**
- Default: `transform: rotate(-0.5deg)` — лёгкий наклон для игривости
- Hover: `translateY(-2px)`, тень 7px — приподнимается
- Active: `translateY(2px)`, тень 2px — вдавливается
- Disabled: `opacity: 0.5`, pointer-events: none

### Comic Panel (карточка)

Основной контейнер контента — имитирует панель комикса.

```css
.comic-panel {
  border: 4px solid #1E293B;
  border-radius: 12px;
  background: #fff;
  box-shadow: 5px 5px 0 #1E293B;
  overflow: hidden;
}
```

### Speech Bubble (пузырь)

Для текста истории в уроке, диалогов персонажей.

- Белый фон, обводка 3px, скругление 18px
- Хвостик вниз (CSS triangles с тенью)
- Может указывать влево или вправо для диалога

### Comic Burst (звезда)

Для акцентных меток: "НОВАЯ!", "ВЕРНО!", "+10 XP".

- Жёлтый фон с clip-path звезда
- Жирный текст, лёгкий наклон
- Используется как badge или overlay

### Progress Bar (полосатый)

Полоска прогресса с анимированными диагональными полосами — чувствуется "движение".

```css
background: repeating-linear-gradient(
  45deg,
  var(--teal), var(--teal) 8px,
  var(--teal-dark) 8px, var(--teal-dark) 16px
);
animation: stripesMove 1s linear infinite;
```

### HUD Bar (шапка урока)

Тёмная панель сверху в стиле игрового интерфейса:
- Слева: кнопка X (закрыть)
- Центр: progress bar
- Справа: XP (оранжевый), сердечки (розовые), streak (жёлтый)

### Badge (значок)

Маленькие цветные метки с обводкой: `badge-orange`, `badge-teal`, `badge-pink`, `badge-lime`, `badge-dark`.

## Анимации

| Имя | Длительность | Easing | Когда |
|---|---|---|---|
| `screenIn` | 0.4s | ease-out | Переход между экранами (fade + slide up) |
| `pulse` | 1.5s | ease | Текущий узел миссии (пульсирует) |
| `glowPulse` | 2s | ease | Свечение текущего узла миссии |
| `shake` | 0.5s | ease | Неправильный ответ — тряска карточки |
| `bounceIn` | 0.6s | ease | Элементы завершения (XP, медали) |
| `floatUp` | 0.5s | ease | Появление статов на экране завершения |
| `slideUp` | 0.3s | ease | Панель обратной связи снизу |
| `confettiFall` | 2-4s | ease-out | Конфетти на экране завершения миссии |
| `stripesMove` | 1s | linear, infinite | Полоски progress bar |
| `spinShield` | 1s | linear | Загрузчик LLM (вращающийся щит) |

### Правила
- Переходы между экранами — fade + slide up (0.4s)
- Feedback панель поднимается снизу (slideUp 0.3s)
- На правильный ответ — bounceIn для burst и цифр XP
- На неправильный — shake всей карточки вопроса
- Конфетти на завершении: 50-60 частиц, случайные цвета из палитры, разная скорость (2-4s)

## Обратная связь (Feedback)

Три состояния — панель поднимается снизу экрана:

| Verdict | Цвет фона | Иконка/текст | XP | Hearts |
|---|---|---|---|---|
| `correct` | `--lime` / зелёный | "ВЕРНО!" | +10 XP | — |
| `partial` | `--yellow` / жёлтый | "ПОЧТИ!" | +5 XP | зависит от настроек |
| `incorrect` | `--red` / красный | "ПРОМАХ!" | — | -1 |

- Текст verdict — крупный, comic-burst стиль
- Ниже — текст пояснения (обычный шрифт)
- Ниже — кнопка "ДАЛЕЕ"
- Для LLM-ошибки: отдельное состояние с кнопкой "Повторить"

## Layout

### Breakpoints

| Название | Ширина | Описание |
|---|---|---|
| Mobile | < 768px | Основной — mobile-first |
| Desktop | >= 1024px | Сайдбары, двухколоночный layout |

### Структура экранов

**Каталог (десктоп):**
- Левый сайдбар (240px): навигация
- Центр: карточки курсов (сетка)
- Правый сайдбар (280px): профиль героя, бейджи

**Каталог (мобайл):**
- HUD-бар сверху (XP, streak, hearts)
- Карточки стопкой
- Нижний таб-бар (иконки)

**Урок (любое устройство):**
- Полноэкранный, минималистичный
- HUD-бар сверху (X | progress | hearts)
- Контент по центру, max-width 600px
- Кнопка действия внизу, фиксирована

| Стандартный термин | В нашем UI |
|---|---|

- Это слой пользовательского UI-copy.
- Во внутренних документах архитектуры, API, БД и тестов сохраняются доменные термины `курс`, `урок`, `каталог`, `профиль`.

## Маскот

Щит-робот из логотипа проекта. Появляется:
- На лендинге (hero-секция)
- На экране онбординга
- У текущего узла в дереве миссий
- На экране завершения миссии (празднует)
- На экране "жизни кончились" (грустный)

Маскот не перегружает UI — появляется в ключевых точках для эмоциональной связи.

## Адаптивность — ключевые правила

1. Все касаемые элементы (кнопки, карточки ответов) >= 44px по высоте
2. Текст в уроке: min 16px, без мелкого шрифта на мобильных
3. Progress bar в HUD: растягивается, min-height 18px
4. Карточки ответов: на мобильном — стопкой (полная ширина), на десктопе — сеткой 2x2
5. Сайдбары скрываются на мобильном, навигация → нижний таб-бар
6. Feedback-панель — всегда снизу, полная ширина
