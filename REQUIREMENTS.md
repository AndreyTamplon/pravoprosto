# Право Просто — Требования

## 1. Платформа

- Web-приложение (не Telegram-бот)
- Адаптивная вёрстка, mobile-first — обязательна корректная работа на мобильных устройствах
- Авторизация только через SSO, без логина/пароля. Поддержка нескольких провайдеров (начинаем с Yandex ID)

## 2. UX/UI: формат Duolingo

Максимально заимствуем проверенный UX/UI Duolingo:
- Короткие уроки (3–5 минут)
- Прогресс-бар в каждом уроке
- Формат урока: сначала блок информации / история (комикс-стиль с картинками), затем задания
- Типы заданий (основные): выбор ответа, свободный ответ (проверяется LLM)
- Типы заданий (дополнительные, на потом): drag-and-drop, true/false
- Ветвление сюжета: в зависимости от ответов история разворачивается по-разному
- Мгновенная обратная связь (`правильно` / `частично правильно` / `неправильно` + объяснение)
- Система жизней / сердечек
- Streak (серия дней подряд)
- XP (очки опыта) за каждый урок
- Уровни и дерево навыков
- Анимации и поощрения за правильные ответы
- Яркий, дружелюбный дизайн для детской аудитории

## 3. Клиентское приложение (ученик)

- Регистрация / авторизация
- Дерево курсов и уроков
- Прохождение уроков в интерактивном формате
- Профиль с прогрессом, XP, streak
- Адаптивный интерфейс (mobile-first)

## 4. Админ-панель (конструктор курсов)

- Создание и редактирование курсов
- Конструктор уроков: добавление заданий разных типов
- Управление порядком уроков и модулей
- Предпросмотр уроков
- Модерация teacher-курсов
- Управление пользователями
- Управление платным platform-контентом: offers, цены, ручное подтверждение оплаты, выдача доступа

## 5. Роль «Учитель»

- Учителя могут регистрироваться и создавать собственные курсы/тесты через конструктор
- Курсы учителей после одобрения админом доступны только по ссылке (не попадают в публичный каталог)
- Публичный каталог — только курсы от платформы (создаются админами)
- Учитель видит прогресс учеников, проходящих его курсы

## 6. Точка расширения: проверка свободных ответов через LLM

- Помимо классических тестовых заданий — задания со свободным ответом, где ученик отвечает своими словами
- Ответ проверяется LLM (оценка корректности, полноты, обратная связь)
- Преимущество перед обычными тестами: невозможно угадать правильный ответ — нужно действительно понимать тему

## 7. Monetization MVP

- На старте платным может быть только контент платформы
- Курсы и уроки учителей по умолчанию бесплатные
- Эквайринг в MVP не подключается
- Ученик должен иметь возможность оставить заявку на платный доступ внутри платформы
- Оплата проходит вне платформы, после чего админ вручную подтверждает оплату и выдает доступ через админку
- Архитектура должна быть готова к последующему подключению online checkout без перелома доменной модели
