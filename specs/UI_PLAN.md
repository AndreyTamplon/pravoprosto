# Право Просто — UI Plan

## Дизайн-система

Источник истины для визуального языка и UI-лексики: [DESIGN_SYSTEM.md](/Users/aatamplon/PycharmProjects/hse/pravoprost/DESIGN_SYSTEM.md).
Этот документ описывает экраны и UX flow. При конфликте токенов, шрифтов, палитры и UI-лексики приоритет у `DESIGN_SYSTEM.md`.

### Референсы
- **Duolingo** — структура урока, дерево прогресса, геймификация, минимализм в уроке
- **Учи.ру** — посадочная страница, каталог курсов, карточки, ролевая навигация

### Стиль
- **Фон:** тёплый кремовый `#FFFBEB`
- **Основной акцент:** `teal` для прогресса и primary UI
- **CTA-акцент:** `orange`
- **Дополнительные:** `pink` для hearts, `yellow` для streak и partial feedback, `lime` для correct states
- **Скругления:** базово `12px`, с comic-border и flat shadow
- **Шрифт:** `Baloo 2`
- **Иллюстрации:** маскот (робот-щит из логотипа) появляется на ключевых экранах
- **Кнопки:** крупные, с comic-outline и offset shadow
- **Mobile-first:** все экраны проектируются сначала под 375px, потом адаптируются на десктоп

### Лексика

- Во внутренних доменных документах используются термины `курс`, `урок`, `каталог`, `профиль`.
- В player-facing UI и маркетинговых экранах рендерится лексика из `DESIGN_SYSTEM.md`:
  - `Курс` -> `Миссия`
  - `Урок` -> `Этап миссии`
  - `Каталог` -> `Штаб героя`
  - `Профиль` -> `Досье героя`

## Экраны — Публичная часть

### P-1. Лендинг (посадочная страница)
**Назначение:** привлечение новых пользователей, объяснение продукта.

**Структура (desktop — два столбца, mobile — один столбец):**
- **Hero-секция:**
  - Слева: заголовок «Право просто — учись защищать себя через игру», подзаголовок, кнопка «Начать» (SSO)
  - Справа: иллюстрация маскота с детьми в стиле комикса
- **Секция «Как это работает»:** 3 шага с иконками (Выбери курс → Пройди историю → Получи навык)
- **Секция «Темы»:** карточки-превью курсов (мошенники, покупки, буллинг) — аналог сетки предметов Учи.ру
- **Секция «Для кого»:** три таба или колонки — Детям / Родителям / Учителям
- **Footer:** ссылки, контакты

### P-2. Экран авторизации (SSO)
**Назначение:** вход через SSO-провайдеров.

**Структура:**
- Логотип + маскот
- Кнопки SSO-провайдеров (Yandex ID первый, остальные ниже)
- Текст «Нет аккаунта? Войдите — он создастся автоматически»
- Без полей логин/пароль

## Экраны — Ученик

### S-1. Выбор роли (первый вход)
**Назначение:** определить роль пользователя при первом входе.

**Структура:**
- «Кто вы?» — три крупные карточки:
  - 🎒 Ученик
  - 👨‍👩‍👧 Родитель
  - 📚 Учитель
- Выбор сохраняется в профиль

### S-2. Онбординг ученика
**Назначение:** объяснить механики за 3–4 экрана.

**Структура (свайпаемые слайды):**
1. Маскот: «Привет! Я помогу тебе разобраться в правах и безопасности»
2. «Проходи истории и принимай решения» (иллюстрация комикса)
3. «Зарабатывай XP и поддерживай streak» (иконки XP, серия, сердечки)
4. «Начнём!» → переход к каталогу курсов

### S-3. Каталог курсов (главный экран)
**Назначение:** выбор курса для прохождения.

**Layout — аналог Duolingo (desktop):**
- **Левый сайдбар:** навигация — Курсы (активный), Профиль, (на mobile — нижний таб-бар)
- **Центр:** единый каталог с секциями:
  - `Курсы платформы`
  - `Курсы по ссылке` (появляется после claim teacher course)
  - внутри секций — вертикальные карточки:
    - Иконка/иллюстрация курса (цветная)
    - Название
    - Краткое описание
    - Прогресс-бар (если начат)
    - Бейдж «Новый» / «Популярный»
- **Правый сайдбар (desktop):** виджеты — streak, XP, сердечки

**Mobile:** нижний таб-бар (Курсы / Профиль), карточки курсов стопкой, streak/XP — в шапке.

### S-4. Дерево уроков (внутри курса)
**Назначение:** показать структуру курса и прогресс.

**Layout — вертикальное дерево (как Duolingo):**
- Вверху: название курса, кнопка «Назад»
- Вертикальная дорожка с узлами-уроками:
  - 🟢 Пройден (зелёная галочка)
  - 🔵 Текущий (подсвечен, пульсирует) — кнопка «Начать»
  - 🔒 Заблокирован (серый, замок)
  - 💳 Платный без доступа / ⏳ ожидает подтверждения оплаты
- Между узлами — пунктирная линия
- Маскот стоит у текущего урока
- Группировка по модулям (разделитель с названием модуля)

Для platform paid lessons:
- `locked_paid` показывает цену и подсказку, что доступ выдается после ручного подтверждения админом
- `awaiting_payment_confirmation` показывает, что заявка уже оформлена и ожидает подтверждения
- CTA для `locked_paid`: кнопка `Оставить заявку`
- CTA для `awaiting_payment_confirmation`: неактивная кнопка `Ожидает подтверждения`

### S-5. Экран урока — блок истории (комикс)
**Назначение:** подача информации перед заданием.

**Layout — полноэкранный, минималистичный (как урок в Duolingo):**
- **Шапка:** кнопка X (закрыть) | прогресс-бар | ❤️ сердечки
- **Центр:**
  - Иллюстрация (комикс-панель) — занимает верхнюю половину
  - Текстовый блок-пузырь (как в комиксе) — ситуация, контекст
  - Может быть диалог персонажей (пузыри слева/справа)
- **Низ:** кнопка «Далее →»

### S-6. Экран урока — задание «Выбор ответа»
**Назначение:** ученик выбирает ответ из вариантов.

**Layout (аналог Duolingo):**
- **Шапка:** X | прогресс-бар | ❤️
- **Центр:**
  - Текст вопроса (крупный шрифт)
  - Опционально: иллюстрация ситуации
  - Карточки вариантов (2–4 штуки, вертикально или сеткой 2×2)
  - Карточка подсвечивается при выборе
- **Низ:** кнопка «Проверить» (неактивна пока не выбран ответ)

### S-7. Экран урока — задание «Свободный ответ»
**Назначение:** ученик пишет ответ своими словами.

**Layout:**
- **Шапка:** X | прогресс-бар | ❤️
- **Центр:**
  - Текст вопроса
  - Опционально: иллюстрация
  - Текстовое поле (multiline, placeholder «Напиши свой ответ...»)
- **Низ:** кнопка «Проверить» (неактивна пока поле пустое)
- После отправки: индикатор загрузки (LLM проверяет), затем обратная связь

### S-8. Экран обратной связи — правильный ответ
**Назначение:** положительное подкрепление.

**Layout:**
- Нижняя панель поднимается (зелёный фон):
  - ✅ «Отлично!»
  - Пояснение (почему это правильно)
  - «+10 XP»
  - Кнопка «Далее»

### S-9. Экран обратной связи — неправильный ответ
**Layout:**
- Нижняя панель (красный/оранжевый фон):
  - ❌ «Не совсем так»
  - Пояснение (правильный ответ и почему)
  - «−1 ❤️»
  - Кнопка «Далее»

### S-9a. Экран обратной связи — частично правильный ответ
**Layout:**
- Нижняя панель (жёлтый фон):
  - ⚠️ «Почти получилось»
  - Пояснение, что в ответе верно и чего не хватило
  - Частичное начисление XP
  - Правило по списанию жизни зависит от настроек геймификации
  - Кнопка «Далее»

### S-10. Экран завершения урока
**Назначение:** итог, поощрение, мотивация продолжать.

**Layout:**
- Маскот с анимацией празднования
- Статистика: XP за урок, точность (%), время
- Streak: «4 дня подряд! 🔥»
- Кнопки: «Продолжить» (к следующему уроку) / «К курсу»

### S-11. Экран «Жизни закончились»
**Layout:**
- Маскот грустный
- «Жизни закончились 💔»
- «Восстановятся через NN минут»
- Таймер обратного отсчёта
- Кнопка «К пройденным урокам» (для практики, без обещания мгновенного восстановления жизней)

### S-12. Профиль ученика
**Layout:**
- Аватар (из SSO или выбранный), имя
- Статистика: общий XP, уровень, текущий streak, пройдено уроков
- Достижения / бейджи (горизонтальный скролл)
- Список начатых курсов с прогрессом

## Экраны — Родитель

### R-1. Кабинет родителя
**Layout:**
- Раздел «Мои дети»
- Карточки детей (имя, аватар, краткая статистика)
- Кнопка «Добавить ребёнка» → создание инвайта/ссылки на привязку

### R-2. Прогресс ребёнка
**Layout:**
- Имя и аватар ребёнка
- Сводка: XP, streak, время в приложении, % правильных ответов
- Список курсов с прогресс-барами
- По каждому курсу можно раскрыть → уроки и результаты

## Экраны — Учитель

### T-0. Онбординг учителя
**Layout:**
- Экран после выбора роли `Учитель`
- Поля: имя, организация/школа
- Кнопка `Продолжить`
- После сохранения -> `T-1. Кабинет учителя`

### T-1. Кабинет учителя
**Layout:**
- «Мои курсы» — список созданных курсов (карточки с названием, статус, кол-во учеников)
- Кнопка «Создать курс»
- Кнопка «Предпросмотр как ученик» для проверки player UI без смены роли

### T-2. Конструктор курса
**Layout:**
- Шапка: название курса (редактируемое), кнопка «Сохранить», кнопка «Отправить на модерацию», кнопка «Поделиться ссылкой» (активна только после публикации)
- Метаданные: описание, целевой возраст
- Список модулей → внутри каждого — список уроков (drag-and-drop для сортировки)
- Кнопки: «+ Модуль», «+ Урок»

### T-3. Конструктор урока
**Layout — центральная панель:**
- Название урока
- Вертикальный список блоков (шагов урока), каждый блок — карточка:
  - **Блок истории:** текстовое поле + загрузка иллюстрации
  - **Выбор ответа:** текст вопроса + варианты (+ / удалить вариант) + отметка правильного + пояснение
  - **Свободный ответ:** текст вопроса + эталонный ответ/критерии для LLM + пояснение
  - **Ветвление:** выбор → какой блок показывать при каждом варианте ответа
- Drag-and-drop для перестановки блоков
- Кнопки: «+ Добавить блок» (выбор типа), «Предпросмотр», «Сохранить»

### T-4. Прогресс учеников (по курсу)
**Layout:**
- Название опубликованного курса, ссылка для расшаривания
- Таблица учеников:
  - Имя | Прогресс (%) | XP | Правильных (%) | Последняя активность
- Клик по ученику → детализация по урокам

### T-4a. Детализация ученика по курсу
**Layout:**
- Имя ученика, краткая статистика
- Таблица уроков: урок, статус, лучший verdict, количество попыток, последняя активность

### T-5. Предпросмотр урока
Открывается в режиме ученика (экраны S-5 – S-10), но с плашкой «Предпросмотр» сверху и кнопкой «Вернуться в редактор».

## Экраны — Админ

### A-1. Админ-панель (дашборд)
**Layout:**
- Боковое меню: Курсы | Модерация | Commerce | Пользователи
- Центр: общая статистика (пользователей, курсов, активность)

### A-2. Управление курсами
**Layout:**
- Список курсов платформы (таблица: название, статус, уроков, учеников)
- Кнопка «Создать курс»
- Конструктор курса/урока — аналогичен учительскому (T-2, T-3)

### A-3. Управление пользователями
**Layout:**
- Таблица: имя, роль, дата регистрации, XP, последняя активность
- Фильтры по роли
- Клик → карточка пользователя с детальной статистикой
- В карточке: действия `Заблокировать`, `Разблокировать`

### A-4. Модерация teacher-курсов
**Layout:**
- Очередь курсов на модерации
- Таблица: курс, автор, дата отправки, статус
- Клик → детали курса, комментарий, кнопки «Одобрить» / «Отклонить»

### A-5. Commerce backoffice
**Layout:**
- Таб `Offers`: список платных platform lesson/course offers, цена, статус
- Таб `Requests`: заявки учеников на получение платного доступа, действия `Создать заказ` / `Отклонить`
- Таб `Orders`: student, offer, сумма, статус, внешний reference
- В карточке заказа: ручное подтверждение оплаты, complimentary access, revoke access
- Отдельный flow выдачи доступа конкретному ученику

## Карта переходов

```
Лендинг (P-1)
  └→ SSO (P-2)
      └→ Выбор роли (S-1) [первый вход]
          ├→ Ученик → Онбординг (S-2) → Каталог (S-3)
          ├→ Родитель → Кабинет (R-1)
          └→ Учитель → Онбординг (T-0) → Кабинет (T-1)

Ученик:
  Каталог (S-3) → Дерево уроков (S-4) → Урок:
    История (S-5) ↔ Задание выбор (S-6) / Задание свободный (S-7)
      → Обратная связь (S-8 / S-9)
      → [жизни = 0] → Жизни кончились (S-11)
      → Завершение (S-10)
  Профиль (S-12)

Родитель:
  Кабинет (R-1) → Прогресс ребёнка (R-2)

Учитель:
  Кабинет (T-1) → Конструктор курса (T-2) → Конструктор урока (T-3)
  Кабинет (T-1) → Прогресс учеников (T-4)
  Предпросмотр (T-5)

Доступ по ссылке:
  Ссылка → [не авторизован?] → SSO (P-2) → Дерево уроков (S-4)
```
